Ask Your Odoo a Question. Get an Answer You Can Trust.

Speak To Your Database lets you ask plain-language questions like “total unpaid invoices,” “sales this month,” or “orders for a customer” — and get answers straight from your live Odoo, each showing its source, a confidence level, and exactly how it was calculated. Read-only and safe by design: no SQL is sent to Odoo, no database credentials are exposed, and nothing in your data is ever changed.

The problem

Your Odoo data is one of your most valuable assets — but getting quick answers from it safely is hard.

Reports take time

Business users constantly need numbers — sales this month, unpaid totals, orders per customer — and end up waiting on custom reports or exports.

Uncontrolled database access is risky

Raw database access should not be shared casually with dashboards or AI tools. Without read-only credentials, network restrictions, and query controls, it can expose your ERP to mistakes, leaks, and uncontrolled queries.

Unrestricted AI over an ERP is risky

Letting a general AI roam freely across ERP data can surface sensitive fields, invent answers, or trigger changes you never intended.

AI answers need to be verifiable

Business users should not have to trust a black-box answer. They need to see where the number came from, what was included, and how it was calculated before acting on it.

The solution

The Odoo Bridge is a small, security-focused module that gives Speak To Your Database a controlled, read-only path into your Odoo — through the Odoo ORM, never raw SQL.

1

Install the Bridge

Add the styd_odoo_bridge module to your self-hosted Odoo 19 and enable it for the connector owner.

2

Connect securely

Generate a token in Odoo, paste it into Speak To Your Database, and select Odoo Bridge as your chat source.

3

Ask & get answers

Ask a supported question and get a clear answer — each with a source, a confidence level, and a “how this was calculated” breakdown.

Version support

This Odoo Bridge module is currently validated for Odoo 19.0 self-hosted and Odoo.sh environments.

For Odoo 15.0, 16.0, 17.0, and 18.0, the Odoo Bridge module is not available yet. In the meantime, Speak To Your Database can connect through a secured, read-only database connection where database access is available. Uncontrolled database access is risky — so this path is meant to be set up carefully, with read-only credentials and proper network restrictions, as a deliberate fallback until the Bridge is validated for each of these Odoo versions.

Bridge module support for older Odoo versions is planned and will be published after controlled validation for each Odoo series.

Security by design

The Bridge is built so the worst case is simply “an allow-listed business record was read” — never a write, a credential disclosure, or an arbitrary query.

  • Read-only ORM access only — list reads, counts, and aggregations.
  • No SQL is sent from Speak To Your Database to Odoo.
  • No write / create / update / delete — there is no write path.
  • Allow-listed models — only a small set of standard business models is readable.
  • Allow-listed fields — only safe field types are returned or filtered.
  • Sensitive credential / security fields are blocked from ever being returned.
  • Token shown once at generation — never displayed again.
  • Hash-at-rest in Odoo — only a SHA-256 hash of the token is stored.
  • Encrypted server-side in Speak To Your Database — used server-side only, never in the browser.
  • No database credentials are ever exposed to Speak To Your Database.
  • Company-scoped access — results follow the connector owner’s Odoo company scope.
  • Safe fallback — unsupported questions return a clear “not supported yet” message, never a guess or a SQL fallback.

Current early access mode: sudo_company_scoped — a company-level boundary. Per-user Odoo record-rule enforcement is not applied yet (see “What’s not supported yet”).

Supported Odoo models

Speak To Your Database is focused on the data behind your everyday revenue, cash, and customer questions. The current bridge reads from a small, deliberate set of standard Odoo business models:

res.partner · customers / contacts
sale.order · sales orders & quotations
account.move · invoices / credit notes
account.move.line · invoice lines
product.template · products
product.product · product variants

What’s not supported yet

Intentionally out of scope for this controlled early access release:

  • Odoo Online (odoo.com-hosted) — custom server modules can’t be installed there.
  • Odoo Bridge marketplace installation for versions older than 19.0 — planned after controlled validation per Odoo series.
  • Arbitrary custom models and custom-module discovery.
  • Non-allow-listed models — anything beyond the six models above.
  • Write / update / delete actions of any kind.
  • Unrestricted SQL-style analytics or free-form query building through the bridge.
  • General-knowledge questions not answerable from your Odoo data.
  • Bridge module support on older Odoo versions unless tested through controlled early-access validation.
  • Full per-user record-rule enforcement — the current bridge is company-scoped (sudo_company_scoped), not per-user ACL/record-rule parity.
